# be64 helper

Uses a value whose big-endian bytes are all ASCII-range, because markspec's
binary matcher reads stdout as UTF-8 chars. The high-byte case is covered by
the rspec suite, which compares bytes directly through Bsm::Generator.

```text file:be64.ebsm
; [be64 0x0102030405060708]
```

```text command
bundle exec ruby exe/ebsm be64.ebsm
```

```binary expected stdout
01 02 03 04 05 06 07 08
```
