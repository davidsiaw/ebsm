# le64 helper

Uses a value whose little-endian bytes are all ASCII-range, because markspec's
binary matcher reads stdout as UTF-8 chars. The high-byte case (e.g.
0x0123456789abcdef) is covered by the rspec suite, which compares bytes
directly through Bsm::Generator.

```text file:le64.ebsm
; [le64 0x0102030405060708]
```

```text command
bundle exec ruby exe/ebsm le64.ebsm
```

```binary expected stdout
08 07 06 05 04 03 02 01
```
